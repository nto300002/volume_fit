## プロダクト定義

Volume Fit は、トレーニング記録をアプリ内だけで分析・完結させるのではなく、外部 AI が解釈しやすいデータへ変換する。

### アプリの責務

- トレーニングの事実を記録する
- 自重運動の負荷を概算する
- ボリューム、RIR 補正値、ハードセット等を計算する
- 過去記録や基準メニューと比較する
- AI が読みやすい入力文を生成する
- AI へ渡した内容を履歴保存する
- AI 提案の次回予定を手動管理する

### 外部 AI の責務

- 記録内容の解釈
- 前回との差の評価
- 疲労・停滞要因の推定
- 自重と外部重量の比較
- 次回メニュー作成
- Notion 保存用文章の生成

### Notion の責務

- 長期保存
- AI 分析結果の蓄積
- 振り返り
- トレーニング知識の整理

## 最重要の価値検証要件

### REQ-CORE-001 入力の負担

トレーニング中に記録が煩雑にならないこと。

最初の検証フロー:

```text
ログイン済み
↓
腕立て伏せを選択
↓
回数を入力
↓
RIR を選択
↓
保存
```

目標:

- 1 セット目の主要操作を 4 操作程度に抑える
- 2 セット目以降は「前セット複製」を利用する
- 入力後、計算結果を即時表示する
- 保存失敗時も入力内容を保持する

### REQ-CORE-002 概算値の理解

推定値を実際の筋肥大効果と誤認させない。

すべての主要な計算画面・AI 出力に次の趣旨を表示する。

> この値はトレーニング比較用の概算です。実際の筋肥大量や筋力向上量を直接示すものではありません。

### REQ-CORE-003 AI 出力の実用性

生成 Markdown を実際に ChatGPT・Claude へ渡し、以下を確認する。

- 記録内容を正しく理解できる
- 概算値を絶対的効果として扱わない
- 次回メニューが具体的に出る
- Notion 用文章がそのまま利用できる
- 毎回追記する指示がないか確認できる

## 利用者・認証要件

### ログイン必須

非ログイン利用は提供しない。

```text
アプリ起動
↓
Firebase 認証状態を確認
├─ 未認証 → ログイン画面
└─ 認証済み → プロフィール確認
```

### 認証方式

- Google ログイン
- メールアドレス + パスワード

### メール確認

メールアドレス登録時の確認メールは必須にしない。登録成功後は直ちに初回プロフィール設定へ進む。

### パスワード再設定

登録済みメールアドレスへパスワード再設定メールを送信できる。

### ログアウト

ログアウト後は、前の利用者のデータを画面へ表示しない。

Firestore のキャッシュが端末内に残っていても、未認証状態ではユーザー固有画面へ入れない。

## プロフィール要件

初回ログイン後に設定する。

| 項目 | 必須 |
|---|---:|
| 表示名 | 任意 |
| 身長 | 任意 |
| 体重 | 必須 |
| トレーニング経験月数 | 任意 |
| 主要目的 | 必須 |
| 使用単位 | kg 固定 |

主要目的:

- 筋肥大
- 筋力向上
- その他

## トレーニング記録要件

### セッション

1 回のトレーニングをセッションとして扱う。

状態:

```text
draft
in_progress
completed
deleted
```

### 外部重量セット

記録項目:

- 重量
- 回数
- RIR
- 結果
- 可動域
- テンポ
- 休憩時間
- メモ

結果:

```text
completed
failed
partial
```

### 自重セット

記録項目:

- 体重
- 自重負荷係数
- 追加重量
- 補助重量
- 回数
- RIR
- 結果
- 可動域
- テンポ
- メモ

### セット複製

直前セットを複製できる。

複製後に変更可能:

- 重量
- 回数
- RIR
- 結果

複製時は新しいセット ID を発行する。

### 保存タイミング

Firestore への保存を以下で行う。

- セット追加
- セット編集確定
- 種目追加
- セッション完了
- 明示的な保存操作
- 画面離脱前の保存処理

最初の Web 縦切りでは、入力途中の毎キー入力保存は行わず、入力確定単位で保存する。

## RIR 要件

RIR は「セット終了時にあと何回できそうだったか」を表す。

UI:

```text
セット終了時、あと何回できそうでしたか？
```

| 表示 | 内部値 |
|---|---:|
| 0 回 | 0 |
| 1 回 | 1 |
| 2 回 | 2 |
| 3 回 | 3 |
| 4 回以上 | 4 |
| 分からない | null |

RPE は MVP では直接入力しない。

## AI 出力要件

### 対象 AI

- ChatGPT
- Claude
- その他

### 目的

- 今日の評価
- 前回比較
- 自重と外部重量の比較
- 筋肥大ボリューム確認
- 次回メニュー作成
- Notion 用整理
- カスタム

### 出力形式

- Markdown
- JSON
- 両方

### AI 共通指示

```text
記録上確認できる事実と推定を分けてください。
不明な情報を創作しないでください。
アプリの計算値は比較用の概算です。
推定ボリュームを筋肥大効果と同一視しないでください。
次回メニューでは、種目、重量または負荷方式、
回数、セット数、目標RIRを示してください。
最後にNotionへ保存しやすいMarkdownを出力してください。
```

### 共有方法

Web:

- クリップボード
- 選択可能テキスト
- JSON ファイル

Android:

- クリップボード
- OS 共有シート
- JSON ファイル

### AI 出力履歴

Cloud Firestore へ保存する。

保存対象:

- 対象セッション
- 比較対象
- 出力目的
- AI サービス
- Markdown
- JSON
- 計算スナップショット
- 計算バージョン
- プロンプトバージョン
- カスタム指示
- AI 回答メモ

AI 回答自体は自動取得しない。

## 次回予定要件

手動登録項目:

- 予定日
- 種目
- 重量または負荷方式
- 回数
- セット数
- 目標 RIR
- メモ
- 元 AI 出力履歴 ID

予定値と実績値は分離する。

予定から新しいセッションを作成する際は、予定値を編集可能な初期値として使用する。
